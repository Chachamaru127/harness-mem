# 業務デモ — 次のAIが実際にやった仕事 (ops-replay-result.md)

これは `examples/senpai-note/ops-handoff-pack.md` の **REPLAY_PROMPT** を、次の担当者の
AI（Claude Code や任意のAIアシスタント）にそのまま貼ったときに返ってくる
**実行結果のサンプル**です。

ポイント: Senpai Note は「引き継ぎを作る」だけで終わらない。出力の REPLAY_PROMPT が
そのまま**次のタスクをAIに実行させる指示書**になる。下記は、引き継ぎを受けたAIが
（人が一切タイプし直さずに）**次の顧客返信を下書きし、対応計画を立てた**もの。

ここが「ドキュメント作成」ではなく「業務の自動化」である証拠になる。

---

## AIが立てた対応計画（人の確認用・30秒）

1. 配送業者ポータルで不在証明の回答有無を確認（未着なら催促）。
2. マネージャー承認の状況を確認（本日午後不在 → 明日朝の前提）。
3. 顧客へ「確認中・次の連絡時期」を、約束しすぎない表現で一報（下書きは下記）。
4. 2条件（不在証明＋承認）が揃い次第、返金処理 → CRMクローズ。月内処理を死守。

※ ルール遵守チェック（REPLAY_PROMPTの "Do not" を自動で守った点）:
- ✅ 注文番号を再度聞いていない
- ✅ 未承認のため「返金します」と確定していない

## なぜこれが効くか（審査ポイント）

- **要約で止まらず、次の実務（顧客返信）まで完了**している → 「自動化」評価
- **新人が踏む地雷を先回りで回避**（注文番号の再質問・未承認返金）→ 「実用性」
- 人がやったのは**承認待ち状態を喋った/貼っただけ**。返信文の起案はAI → 工数削減
