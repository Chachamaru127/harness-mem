# ベンチマーク測定フレームワーク設計書

> **バージョン**: 1.0
> **作成日**: 2026-03-04
> **対象**: §32 ベンチマーク信頼性改革

---

## 1. 目的と背景

本設計書は、harness-mem の検索品質を客観的・再現可能に測定するための標準を定義する。
競合比較（OMEGA 95.4%、Mastra 94.87%）との公正な比較条件の明文化と、回帰防止ゲートの確立が主目的である。

---

## 2. 評価レイヤーの定義

harness-mem の評価は **2 つのレイヤー** に明確に分ける。

### レイヤー A: 検索レイヤー（IR メトリクス）

**「正しい記憶が Top-K に含まれるか」を測る。**

- LLM 推論を含まない、純粋な検索精度の評価
- ベクトル検索 + FTS 検索 + リランカーの組み合わせ精度を対象とする
- 本フレームワークの主計測対象

| メトリクス | 説明 | 計算式 | 用途 |
|---|---|---|---|
| **Recall@10** | 正解が Top-10 内に含まれる割合 | `hits / |relevant|` | 網羅性の評価 |
| **Precision@10** | Top-10 のうち正解の割合 | `hits / 10` | ノイズ排除の評価 |
| **MRR** (Mean Reciprocal Rank) | 正解の最初のランクの逆数の平均 | `Σ(1/rank) / N` | 1位精度の評価 |
| **NDCG@10** (Normalized DCG) | ランク付き関連度の正規化スコア | `DCG / IDCG` | ランキング品質の評価 |

### レイヤー B: Answer F1（エンドツーエンド）

**「正しい答えを文字列として生成できるか」を測る。**

- locomo-harness-adapter.ts が行う候補選択・答え抽出・正規化を含む
- LLM 非依存（ルールベース抽出のみ）
- 競合比較に使用するメトリクス

| メトリクス | 説明 | 計算式 |
|---|---|---|
| **EM** (Exact Match) | 正規化後に完全一致する割合 | `(prediction == answer) ? 1 : 0` |
| **F1** (Token F1) | トークン単位の Precision と Recall の調和平均 | `2 * P * R / (P + R)` |

## 4. 競合スコアとの比較条件

### 4.1 比較対象と注意点

| 競合 | 公表スコア | 測定条件 | 比較時の注意 |
|---|---|---|---|
| OMEGA | 95.4% | 不明（自社測定） | 測定条件が異なる可能性が高い |
| Mastra | 94.87% | 不明（自社測定） | 同上 |
| harness-mem | 測定中 | 本設計書に基づく | 検索レイヤーのみ |

### 4.2 比較条件の明示義務

競合比較表を作成・更新する際は、必ず以下を明記すること:

```
【測定条件】
- 評価レイヤー: 検索レイヤーのみ / Answer F1 込み
- データセット: locomo-120 (40 samples, 120 QA)
- メトリクス: recall@10 / MRR / NDCG@10 / F1
- 測定日: YYYY-MM-DD
- 競合スコアの出典: 自社公表値（独立検証なし）
```

## 5. 各質問タイプ別メトリクス定義

### cat-1: 単純事実 (Single-hop Factual)

- **主要メトリクス**: EM, F1
- **目標 F1**: 0.35+（baseline: 0.300）
- **評価方法**: 会話中の単一事実の直接抽出精度

### cat-2: 時間軸 (Temporal)

- **主要メトリクス**: F1（時間表現の部分一致を重視）
- **目標 F1**: 0.15+（baseline: 0.099）
- **評価方法**: 日時・期間・順序の正確な抽出
- **注意**: EM が低くなりやすい（表現の揺れが多い）

### cat-3: 多段推論 (Multi-hop / Counterfactual)

- **主要メトリクス**: F1（厳密 EM は困難）
- **目標 F1**: 0.20+（baseline: 0.114）
- **評価方法**: 複数証拠の連鎖・反事実的推論の妥当性

### cat-4: 知識更新 (Knowledge Update)

- **主要メトリクス**: EM, F1
- **目標 F1**: 0.30+（baseline: 0.204）
- **評価方法**: 更新後の情報が古い情報より優先されるか

## 6. 統計的妥当性の要件

### 6.1 最低サンプル数

| 用途 | 最低サンプル数 | 根拠 |
|---|---|---|
| 開発中スモークテスト | 10 QA | 高速フィードバック |
| 回帰ゲート（CI） | 120 QA（locomo-120 全件） | 統計的安定性 |
| 競合比較報告 | 120 QA 以上 | 信頼区間の確保 |

### 6.2 標準誤差と信頼区間

120 サンプルでの F1 スコアの標準誤差（σ/√n）の目安:

- F1 ≈ 0.18、σ ≈ 0.20 の場合: SE ≈ 0.018（±1.8 pp）
- **回帰ゲート閾値**: 直前 baseline から -3 pp 以上の低下で警告

### 6.3 回帰ゲートの運用ルール

```
警告条件: overall F1 < (baseline_f1 - 0.03)
エラー条件: overall F1 < (baseline_f1 - 0.05)
```

CI 実行時にエラー条件に達した場合、PR マージをブロックする。

## 7. 「検索レイヤーのみ」vs「LLM 推論込み」の区分

### 検索レイヤーのみ（本フレームワークの主対象）

```
[会話 ingest] → [search(query)] → [Top-K IDs] → Recall/MRR/NDCG
```

- ルールベース答え抽出（locomo-harness-adapter.ts）を使用
- LLM API 呼び出しなし
- 再現可能・コスト不要

### LLM 推論込み（将来対応）

```
[会話 ingest] → [search(query)] → [Top-K コンテキスト] → [LLM] → [答え] → EM/F1
```

- 外部 LLM API（Claude, GPT-4 など）が必要
- コストと再現性の問題がある
- §32 スコープ外（明示的な要件追加まで実装しない）
