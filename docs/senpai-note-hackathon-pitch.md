# Senpai Note — AI Ops Hackathon 発表構成

対象: AI Ops Hackathon @関西（2026-05-31, 大阪 Blooming Camp）
テーマ: 普段の効率化しきれていない業務を AI Agent で自動化
配点: 実用性 40 / AI活用度 30 / 完成度・プレゼン 30
発表枠: 全体約60分（チーム数で割ると 1 チームおおよそ 3〜5 分想定）。本構成は
**3 分版**を基準にし、90 秒に圧縮できる短縮ガイドも併記する。

> 一言ピッチ: **「その人しか分からない業務を、誰でも 30 秒で引き継げる
> カード＋手順＋AI Agent 指示書に、Claude Code がその場で変換する。」**

---

## スライド構成（3 分・全6枚）

### S1. タイトル / 一言価値（10秒）
- Senpai Note — 業務の属人化を、引き継ぎパックに変える AI Agent
- 「作業ログ → HANDOFF_CARD / RUNBOOK / REPLAY_PROMPT」
- 採点狙い: 第一印象で「何の業務課題か」を即伝える（完成度・プレゼン）

### S2. 課題（30秒）— 実用性の布石
- 「担当者が休む / 辞める / 異動するたびに、業務が止まる・聞き直しが発生する」
- 引き継ぎは口頭やSlackやNotionに散らばり、次の人は同じ調査・同じ失敗・同じ質問を繰り返す
- 一文: **属人化は“情報が無い”のではなく、“次の人が動ける形になっていない”問題**
- 採点狙い: 実用性40 = 誰の・どの業務の・いくらの損失か

### S3. 解決策（30秒）— AI活用度の布石
- 雑な作業ログを入力するだけで、Claude Code の Skill が 3 点セットを生成:
  - **HANDOFF_CARD**: 30秒で読む引き継ぎカード（開始点・結論・やってはいけない事・次の一手）
  - **RUNBOOK**: 同種業務の再利用手順
  - **REPLAY_PROMPT**: 次の担当者の AI Agent にそのまま貼れる指示書
- ポイント: **出力がそのまま次の自動化の入力になる**（replay prompt → AI Agent）
- 採点狙い: AI活用度30 = Claude Code の Skill 機能を“業務の型”に使っている

### S4. ライブデモ（80秒）— 完成度の核 ★勝負どころ
**3 ステップで「集める→変換する→AIが実行する」を一気通貫で見せる。**

1. **集める（声 or 貼る / どちらでも可）**
   - 入力は `examples/senpai-note/ops-voice-input.md`（喋り起こし）か `ops-demo-session.md`（手元メモ）
   - 声を使う場合: その場で「今日の返金の件なんですけど…」と喋り → 文字起こし
   - **声が無くてもよい**（VoiceOS はオプション。使えれば上振れ、使わなくても成立）
2. **変換する（⏱️ ストップウォッチを回す）**
   - 「**このセッションを Senpai Note にして。次の人が30秒で引き継げる形に**」
   - HANDOFF_CARD の「Do not repeat（**注文番号を再度聞かない／未承認で返金しない**）」を指差す
   - 「引き継ぎ資料作成 **約30分 → 約30秒**」とタイムを読み上げる
3. **AIが実行する（←ここが"自動化"の証明）**
   - 生成された **REPLAY_PROMPT を別の AI に貼る**
   - AI が `ops-replay-result.md` 相当の**顧客返信ドラフトと対応計画を自動生成**
   - 「人がやったのは喋った/貼っただけ。返信の起案はAI」と締める
- 採点狙い: 完成度30（動く）+ AI活用度30（AIが次の実務を実行）+ 実用性40（時間削減）

### S5. なぜ効くか（30秒）— 差別化と「これは自動化だ」
- **ドキュメント作成ではなく業務の自動化**: ①引き継ぎ作成という"効率化しきれない業務"
  そのものを自動化し、②REPLAY_PROMPT で**次の実務（顧客返信）までAIが実行**する
- **単なる要約ではない**: 「次の人が動けること」に最適化（開始点・禁止事項・復旧手順）
- **source 明示**: どの記憶/ログから作ったかを必ず表示 → 嘘をつかない
- **ローカル完結 / 記憶daemon不要**: 貼ったログ・喋った内容だけで動く → 今日から現場で
- 採点狙い: 実用性40 + AI活用度30 = 自動化の実体・安全性・現場導入性

### S6. 横展開 / クローズ（20秒）
- 経理の月次締め、サポート、採用、問い合わせ…**属人化する全業務に同じ型**
- 「作業ログを、チームの“先輩の知恵”資産に変える」
- （任意）将来: 生成 runbook を手順メモリに蓄積 / チーム共有

## トーク台本（3分・読み上げ用）

1. (S1) 「Senpai Note です。“その人しか分からない業務”を、誰でも引き継げる形に
   変える AI Agent を作りました。」
2. (S2) 「業務が止まる原因は、情報が無いからではありません。引き継ぎが“次の人が
   動ける形”になっていないからです。口頭やチャットに散らばり、次の人は同じ調査と
   同じ失敗を繰り返します。」
3. (S3) 「Senpai Note は、雑な作業ログを入れるだけで、Claude Code の Skill が 3 点を
   出します。30秒で読む引き継ぎカード、再利用できる手順書、そして次の担当者の
   AI Agent にそのまま貼れる指示書です。」
4. (S4) 「実際に見せます。これは返金対応の作業ログ——今この場で喋ってもいいし、
   メモを貼ってもいい。（ストップウォッチ）『このセッションを Senpai Note にして』
   ……はい、約30秒。普段なら引き継ぎ資料に30分かかる作業です。注目は“やっては
   いけないこと”。注文番号を再度聞かない、未承認で返金しない。新人が踏む地雷を
   先回りで止めます。そして——この指示書を別の AI に貼ると、**次の顧客返信を
   AIが自分で書き上げます**。人がやったのは喋っただけ。」
5. (S5) 「ここがポイント。これはドキュメント作成ではなく“自動化”です。引き継ぎ
   作成そのものを自動化し、次の実務までAIが実行する。要約ではなく“次の人が動ける”
   形に最適化し、出典も必ず明示。ローカルで完結し、喋った内容だけで動くので、今日
   から現場で使えます。」
6. (S6) 「経理、サポート、採用——属人化するあらゆる業務に同じ型が効きます。作業
   ログを、チームの先輩の知恵に変える。Senpai Note でした。」

## 90秒短縮版（質疑が短い/枠が小さい場合）
- S1（5秒）→ S2 を 1 文（15秒）→ S4 ライブデモ（55秒, ただし**ステップ3のAI実行は
  必ず残す**）→ S5 「自動化だ」の 1 文（10秒）→ S6 クローズ 1 文（5秒）。
- 削るなら S3 とストップウォッチ。**「AIが次の実務を実行する」場面だけは絶対に削らない**
  （ここが他チームとの差＝自動化の証明）。

## 当日2時間ビルドの使い方（実装は既に完了している前提）
機能追加ではなく「疑いを消す1点」に投資する。優先順:
1. **AI実行チェーンの動作確認**（REPLAY_PROMPT → 別AIで顧客返信が出る）を本番環境で通す
2. **ストップウォッチ演出**のリハ（30分→30秒の対比を口で言えるように）
3. （上振れ）**VoiceOS の声入力**を会場で通せるか試す。通れば S4 ステップ1を声に。
   不安定なら `ops-voice-input.md` を貼る運用に切り替え（判断は本番30分前まで）。
4. フォールバック確認（`ops-handoff-pack.md` / `ops-replay-result.md` を即出せる）

## デモが詰まった時のフォールバック
- ネット/記憶daemonに依存しない: 入力は `ops-demo-session.md`、想定出力は
  `ops-handoff-pack.md` に用意済み。ライブ生成が失敗しても完成形を即提示できる。
- スクショ/印刷を 1 枚持っておく（HANDOFF_CARD の「Do not repeat」が一番刺さる）。

## 質疑想定（Q&A）
- Q: ただの要約と何が違う? → A: 開始点・禁止事項・復旧手順・次の一手まで“動ける形”で出す。Quality bar で「少なくとも1つの失敗を防ぐ」ことを必須化。
- Q: 嘘を生成しない? → A: 必ず `source:` を出し、記憶が無い時は user-provided と明記。根拠の無い断定をしない設計。
- Q: 非エンジニアでも使える? → A: 入力は雑メモでOK、記憶daemon不要。トリガーフレーズを話すだけで発火。
- Q: 既存ツール（Notion等）との違い? → A: 保管ではなく“次の人が動ける形への変換”。出力が AI Agent の指示書になる点が新しい。
- Q: それは業務の自動化ではなく“ドキュメント作成”では? → A: ①引き継ぎ作成という業務そのものを自動化し、②REPLAY_PROMPT で次の実務（顧客返信）までAIが実行する。デモのステップ3で実演済み。
- Q: 結局その作業ログ、自分で書くのでは? → A: 整った文章は不要。雑メモでも、**喋った内容（VoiceOS等の文字起こし）でも**動く。入力のハードルを下げてある。
- Q: VoiceOS は必須? → A: いいえ。声は上振れ要素で、無くても貼ったログで完結する（依存しない設計）。
