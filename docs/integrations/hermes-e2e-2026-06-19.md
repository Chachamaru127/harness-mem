### Step 3 — Semantic-equivalent smoke via HTTP API

plugin が呼ぶ HarnessMemClient.record_event は最終的に harness-mem の
`POST /v1/events/record` を叩く。Python deps を bypass して同じ contract を
HTTP で直接実行することで、plugin の動作 path 末端を観測した。

```bash
# session_start event
curl -sS -X POST http://127.0.0.1:37888/v1/events/record \
  -H "content-type: application/json" \
  -d '{"event":{"event_id":"hermes-e2e-smoke-001","platform":"hermes",
       "project":"hermes-e2e-smoke","session_id":"hermes-e2e-2026-06-19",
       "event_type":"session_start",
       "payload":{"content":"S112-005 E2E smoke via HTTP API"},
       "tags":["hermes-e2e","s112-005"],"privacy_tags":[]}}'

# session_end event
curl -sS -X POST http://127.0.0.1:37888/v1/events/record \
  -H "content-type: application/json" \
  -d '{"event":{"event_id":"hermes-e2e-smoke-002","platform":"hermes",
       "project":"hermes-e2e-smoke","session_id":"hermes-e2e-2026-06-19",
       "event_type":"session_end",
       "payload":{"content":"S112-005 E2E smoke session_end via HTTP API"},
       "tags":["hermes-e2e","s112-005"],"privacy_tags":[]}}'

# Verify observation via search
curl -sS -X POST http://127.0.0.1:37888/v1/search \
  -H "content-type: application/json" \
  -d '{"query":"S112-005 E2E smoke","project":"hermes-e2e-smoke","limit":10}'
```

**結果:** PASS。

| obs_id | content |
|---|---|
| `obs_hermes-e2e-smoke-001` | S112-005 E2E smoke via HTTP API |
| `obs_hermes-e2e-smoke-002` | S112-005 E2E smoke session_end via HTTP API |

両 event とも `harness_mem_search` (HTTP `/v1/search`) で `project=hermes-e2e-smoke` フィルタ越しに **観測可能**。harness-mem 側 contract (event 記録 → search で観測) は実機 granite live default で稼働。
