# 業務デモ — 声で残した作業ログ (ops-voice-input.md)

VoiceOS などで「今日やったこと」を喋って文字起こしした想定の入力サンプル。
`ops-demo-session.md`（整理された手元メモ）と**同じ案件**を、より生々しい
話し言葉で表現している。

使い方は2通り、どちらでもデモが成立する:

- **声を使う場合:** この内容を声に出して読む → VoiceOS等で文字起こし → Senpai Note に渡す
- **声を使わない場合:** この文字起こしテキストをそのまま Senpai Note に貼る

どちらでも出力は `ops-handoff-pack.md` に近い handoff pack になる。
（＝ 声入力は"上振れ"であって、無くてもデモは完結する）

えーっと、今日の返金の件なんですけど。注文番号 DEMO-4821 のお客様、昨日からけっこう
怒ってて。朝イチで一次対応の子が「返金できます」って口頭で言っちゃってるんですよね、
これがまずちょっとマズくて。まだ確定してないので。

で、金額が3万円超えてるんで、これマネージャー承認いるやつなんですよ。注文番号は
もう聞いてあります。ていうか3回くらい聞いちゃってるんで、もう二度と聞かないように。

在庫システム見ると「発送済み」になってるんですけど、配送業者の追跡だと「持ち戻り」に
なってて。たぶん住所不備ですね。返金ポリシー的には、商品未着で配送業者の不在証明が
あればフル返金OKなんで、いま配送業者に不在証明の発行をお願いしてます。だいたい翌営業日
には返ってくるかな。

マネージャーが今日の午後いないんで、承認は明日の朝イチでもらう感じで。CRMのステータスは
「対応中」のままにしといてください、クローズしないで。あと月末が明後日なんで、できれば
今月中に返金処理まで終わらせると経理の締めに間に合います。

やってほしいのは、不在証明が来てるか確認するのと、お客様に「いま確認中です、いつ頃
また連絡します」っていう返信を、約束しすぎない感じで出しといてほしい、って感じです。
